## 1. エグゼクティブサマリー

本提案は、茨城県立下妻第一高等学校様が掲げるミッション「**日常の行動データを活用し、生徒一人ひとりの非認知能力を可視化せよ**」に対する、**個人情報保護を最優先**に設計した最終的な答えです。

私たちは、教育現場の最も重要な懸念である「プライバシー保護」と「教師の実用性」を両立させる、統合的な生徒理解・指導支援プラットフォーム「**チームコンパス**」を提案します。このシステムは、以下の3つの特徴により、教育現場の課題を安全かつ効果的に解決します。

1.  **厳格なAI活用範囲**: AIの活用を、**個人情報に触れない領域（音声の文字起こし、生徒向けゲーミフィケーション）のみ**に限定。生徒の記録の要約や分析には一切AIを使用せず、プライバシーを徹底的に保護します。

2.  **音声入力と従来型記録管理**: 教師がスマホに話しかけるだけで記録できる**音声入力機能**で、入力の負担を劇的に軽減。記録の整理は、AIに頼らない**従来型の検索・手動タグ付け機能**で、教師が直感的に操作できる実用的な仕組みを提供します。

3.  **データに基づく安全な指導支援**: 診断データと教師が記録した具体的なエピソードを、**AIを介さずに**レポートとして出力。三者面談や進路相談において、教師はデータに基づきつつも、自身の専門的な判断を加えた、質の高い対話を実現できます。

「チームコンパス」は、最先端技術の利便性を享受しつつも、個人情報保護という教育現場の絶対的な要請に応える、**信頼と革新を両立した唯一のソリューション**です。この安全なプラットフォームこそが、貴校の教育理念の実現に貢献できると確信しております。

## 2. 設計思想：プライバシー・バイ・デザイン

### 2.1. 課題：テクノロジーの利便性とプライバシーの懸念

AIなどの先端技術は、教育に革命をもたらす可能性を秘めています。しかし、その一方で、生徒の行動や内面に関する機微な情報をAIに分析させることは、**個人情報の学習利用やプライバシー侵害**といった重大な懸念を生じさせます。この懸念は、学校・保護者にとって、新しいテクノロジー導入の最大の障壁となります。

### 2.2. 私たちの答え：AI活用の明確な線引き

「チームコンパス」は、この課題を解決するため、「**プライバシー・バイ・デザイン**」の思想に基づき、企画・設計段階から個人情報保護を最優先に組み込んでいます。

AIの活用範囲を、個人情報に触れない領域に厳格に制限し、教師の記録管理は従来型の機能で実現します。

| 領域 | AI活用の可否 | 具体的な機能 |
| :--- | :--- | :--- |
| **音声の文字起こし** | **〇 許可** | Whisper APIを利用。ただし、音声データは即時削除し、モデルの学習には一切使用しない設定。 |
| **生徒向けゲーミフィケーション** | **〇 許可** | 個人を特定できない診断結果の数値のみを使い、AIがキャラクターや成長メッセージを生成。 |
| **記録の要約・分析・タグ付け** | **× 禁止** | 教師が記録した生徒の具体的な行動や発言は、一切AIに送信しない。 |
| **面談資料の生成** | **× 禁止** | AIは使わず、システムが診断データと記録をテンプレートに当てはめてレポートを生成。 |

詳細は添付の「**AI活用範囲の再定義とプライバシー設計書**」をご参照ください。

## 3. システム詳細（プライバシー優先版）

### 3.1. 生徒向けUX：「キャラクターで知る、自分と仲間」

（※このセクションは、AIが個人情報に触れないため、前回の提案から変更ありません）

生徒の参加意欲を引き出すため、楽しく直感的なインターフェースを提供します。

-   **診断体験**: 10分間のWebアンケートに答えると、6つの能力バランスに基づき、「今のあなたに近い成長キャラクター」が表示されます。
-   **自己理解**: 自分の強み・弱みを、高校生に響く平易な言葉で理解します。
-   **多様性理解**: 相性診断を通じて、他者と協働する大切さを学びます。
-   **成長の示唆**: 「キャラは成長と共に変わるかも！」というメッセージで、成長意欲を刺激します。

### 3.2. 教師向けツール：「安全に、効率よく、深く」

個人情報保護を徹底しながら、教師の実用性を最大限に高める機能を提供します。

#### ① 音声入力による観察記録

-   **概要**: 教師がスマホアプリに話しかけるだけで、生徒の観察記録をその場で作成できます。
-   **技術**: OpenAIの**Whisper API**を活用し、高精度な文字起こしを実現。音声データは文字起こし後に**即座に削除**され、モデルの学習にも使用されません。
-   **価値**: 記録の負担を劇的に軽減し、記憶が鮮明なうちに具体的なエピソードを蓄積できます。

#### ② 従来型の記録管理機能

-   **概要**: AIに頼らず、教師が直感的に操作できる、シンプルかつ強力な記録管理機能です。
-   **機能**: 
    -   **手動タグ付け**: 「リーダーシップ」など、6つの能力タグをワンクリックで付与。
    -   **キーワード検索**: 記録のテキスト全文を対象に、高速なキーワード検索が可能。
    -   **フィルタリング**: タグ、日付、生徒で記録を自由に絞り込み。
-   **価値**: 教師は、自身の専門的な判断に基づき、記録を整理・活用できます。

詳細は添付の「**従来型記録管理機能 詳細設計書**」をご参照ください。

#### ③ テンプレート方式による面談資料生成

-   **概要**: 三者面談や進路相談の前に、ボタン一つで診断データと記録を統合したPDFレポートを生成します。
-   **技術**: AIは一切使用せず、システムがデータをあらかじめ用意された**テンプレートに当てはめて**生成します。
-   **価値**: 教師は、準備時間を大幅に削減できると同時に、AIによる意図しない解釈が入り込むリスクを完全に排除できます。

## 4. 具体的な利用シーン

（※このセクションは、教師の体験価値を示すものであり、前回の提案から変更ありません）

「チームコンパス」は、教育現場の様々な場面で活用できます。

### シーン1：三者面談

-   **Before**: 記憶と手元の成績資料だけが頼り。抽象的な説明になりがち。
-   **After**: システムが生成したレポートを提示。「田中さんは、文化祭で実行委員長として活躍し、リーダーシップが20点向上しました」と、**データと具体的なエピソード**で成長を説明。保護者の納得度が格段に向上します。

### シーン2：進路相談

-   **Before**: 生徒の自己申告と成績に頼ったアドバイス。
-   **After**: 能力データと日々の記録から、「君は創造性が高いから、企画職も向いているかもね」と、**生徒自身も気づいていない可能性**を提示。多角的な視点でキャリア選択を支援します。

## 5. 技術仕様と導入計画

### 5.1. 技術スタック

本システムは、セキュリティと信頼性を最重要視し、全てのインフラを**Microsoft Azure**で構築します。

-   **Webアプリケーション**: Azure App Service
-   **データベース**: Azure SQL Database (TDEによる暗号化)
-   **認証基盤**: Azure AD B2C
-   **AI機能**: Azure OpenAI Service (Whisper API)
    -   **※モデルの学習には一切使用しない設定を適用**

### 5.2. 導入計画（MVP：4ヶ月）

最短4ヶ月で、貴校に最適化されたMVPを導入します。

-   **1ヶ月目**: 要件定義、Azure環境設計・構築
-   **2ヶ月目**: 開発（診断機能、音声記録機能）
-   **3ヶ月目**: 開発（記録管理機能、教師向け画面）、テスト
-   **4ヶ月目**: 導入、教師向け研修、運用開始
