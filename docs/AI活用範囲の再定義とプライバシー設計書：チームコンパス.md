# AI活用範囲の再定義とプライバシー設計書：チームコンパス

## 1. 設計思想

**「個人情報保護を最優先とし、AIは必要最小限の範囲でのみ活用する」**

教育現場では、生徒の個人情報保護が最も重要な課題です。特に、AIによる学習や分析が、生徒のプライバシーを侵害するのではないかという懸念は、学校側・保護者側の双方から強く指摘されています。

そこで、「チームコンパス」では、AIの活用範囲を**個人情報に触れない領域のみ**に厳格に制限し、教師の記録管理は従来型の検索・タグ付け機能で実現する、プライバシー重視の設計を採用します。

---

## 2. AI活用の明確な線引き

### 2.1. AI活用を**許可する**領域（個人情報に触れない）

| 領域 | 具体的な機能 | 使用技術 | 理由 |
|---|---|---|---|
| **音声の文字起こし** | 教師が録音した音声を、テキストに変換 | Whisper API | 音声データは即座に削除し、テキストのみを保存。Azure OpenAI Serviceの「学習に使用しない」設定を有効化。 |
| **生徒向けゲーミフィケーション** | 診断結果に基づき、キャラクターや成長メッセージを生成 | GPT-4 | 個人を特定できない、一般的な診断結果のみを入力。生徒の氏名や詳細な記録は含めない。 |
| **診断結果の一般的な解説** | 「リーダーシップが高い人の特徴」といった、汎用的な解説文を生成 | GPT-4 | 個人情報を含まない、教育的なコンテンツの生成のみ。 |

### 2.2. AI活用を**禁止する**領域（個人情報を含む）

| 領域 | 従来の提案 | 新しい方針 | 理由 |
|---|---|---|---|
| **記録の要約** | GPT-4が自動で要約 | **教師が手動で編集** | 記録には生徒の詳細な行動や発言が含まれるため、AIに送信しない。 |
| **記録のタグ付け** | GPT-4が自動でタグを付与 | **教師が手動でタグを選択** | 記録の内容をAIに分析させない。 |
| **成長トレンド分析** | GPT-4が記録を横断的に分析 | **将来機能として検討** | 個人情報を含む記録を、AIに大量に送信することになるため、MVPでは実装しない。 |
| **面談資料の自動生成** | GPT-4が記録を統合してレポート生成 | **テンプレート方式で生成** | AIを使わず、システムが診断データと記録をテンプレートに当てはめて生成。 |

## 3. 音声文字起こしのプライバシー保護設計

音声文字起こしは、唯一AIを活用する個人情報関連機能です。そのため、以下の厳格な保護措置を講じます。

### 3.1. データフロー

1.  **録音**: 教師がスマホアプリで音声を録音。
2.  **暗号化送信**: 音声データは、TLS 1.3で暗号化され、Azure OpenAI Serviceに送信。
3.  **文字起こし**: Whisper APIが音声をテキストに変換。
4.  **音声データの即時削除**: 文字起こし完了後、音声データは**即座に削除**され、サーバーには一切保存されない。
5.  **テキストの保存**: 文字起こしされたテキストのみが、Azure SQL Databaseに保存される。

### 3.2. Azure OpenAI Serviceの設定

-   **学習に使用しない設定を有効化**: Azure OpenAI Serviceでは、顧客データをモデルの学習に使用しない設定が可能です。この設定を有効化し、生徒の音声データが学習に使われないことを保証します。
-   **データ保持期間の最小化**: Azure OpenAI Serviceでは、処理後のデータ保持期間を最小限（30日以内）に設定できます。この設定を適用します。

### 3.3. 保護者・学校への説明

-   **同意書の取得**: 音声文字起こし機能の使用前に、保護者と学校に対して、データの取り扱い方法を明記した同意書を取得します。
-   **透明性の確保**: システムの設定画面で、「音声データは文字起こし後に即座に削除されます」と明記し、透明性を確保します。

## 4. 従来型記録管理機能の設計

AIを使わずに、教師が記録を効率的に管理できる仕組みを提供します。

### 4.1. 記録の編集機能

-   **文字起こし後の編集**: 音声から文字起こしされたテキストは、教師が自由に編集できます。不要な部分を削除したり、要点を強調したりできます。
-   **手動要約**: 教師が、記録の冒頭に「要約」欄を設け、自分の言葉で要点をまとめます。

### 4.2. 手動タグ付け機能

-   **プリセットタグ**: 「リーダーシップ」「協調性」「創造性」など、6つの能力に対応したタグをプリセットで用意。
-   **ワンクリック選択**: 教師は、記録保存時に、関連するタグをワンクリックで選択できます。

### 4.3. 検索・フィルタリング機能

-   **キーワード検索**: 記録のテキスト全文を対象に、キーワード検索が可能。
-   **タグフィルタ**: 「リーダーシップに関する記録のみ表示」といった、タグによるフィルタリングが可能。
-   **日付フィルタ**: 「過去1ヶ月の記録のみ表示」といった、日付によるフィルタリングが可能。

### 4.4. 面談資料のテンプレート生成

-   **AIを使わない生成**: システムが、診断データと記録を、あらかじめ用意されたテンプレートに当てはめて、PDFレポートを生成します。
-   **教師による最終調整**: 生成されたレポートは、教師が最終的に確認・編集してから使用します。

## 5. 生徒向けゲーミフィケーションでのAI活用

生徒向けの機能では、個人情報に触れない範囲で、積極的にAIを活用します。

### 5.1. キャラクター診断のパーソナライズ

-   **入力データ**: 診断結果の6つの能力スコア（数値のみ）
-   **AI処理**: GPT-4が、スコアに基づき、生徒に響くメッセージを生成。例：「君は創造性が高いから、新しいアイデアを考えるのが得意だね！」
-   **個人情報の非含有**: 生徒の氏名、クラス、記録内容は一切含めない。

### 5.2. 成長メッセージの生成

-   **入力データ**: 前回と今回の診断スコアの差分（数値のみ）
-   **AI処理**: GPT-4が、成長を称賛するメッセージを生成。例：「リーダーシップが10点も伸びたね！この調子で頑張ろう！」
-   **個人情報の非含有**: 生徒の氏名、具体的なエピソードは含めない。

## 6. プライバシー保護の技術的保証

### 6.1. データの暗号化

-   **通信の暗号化**: 全ての通信は、TLS 1.3で暗号化されます。
-   **データベースの暗号化**: Azure SQL Databaseの「Transparent Data Encryption (TDE)」を有効化し、保存データを暗号化します。

### 6.2. アクセス制御

-   **役割ベースのアクセス制御（RBAC）**: 教師は自分が担当する生徒のデータのみにアクセス可能。管理者は全データにアクセス可能。
-   **監査ログ**: 全てのデータアクセスは、監査ログに記録され、不正アクセスを検知できます。

### 6.3. データの保持期間

-   **卒業後のデータ削除**: 生徒が卒業した後、一定期間（例：1年）経過後に、データを自動で削除します。
-   **保護者の削除要請**: 保護者からの要請があれば、いつでもデータを削除できます。

---

## 7. 結論

「チームコンパス」は、**個人情報保護を最優先**としながらも、音声入力による記録の負担軽減と、生徒向けゲーミフィケーションによるエンゲージメント向上を実現します。

AIの活用範囲を厳格に制限し、教師の記録管理は従来型の検索・タグ付け機能で実現することで、学校側・保護者側の懸念を払拭し、安心して導入いただけるシステムを提供します。

これこそが、教育現場が真に求める、**信頼と革新の両立**です。
